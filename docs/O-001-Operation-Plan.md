## 1. 운용 철학: MAI Universe "기여와 수익"

> "돈만 쫓으면 얕아지고, 기여만 하면 지속이 안 된다.  
> 진짜 가치를 만드는 건 둘의 교차점이다." — SOUL.md

M.AI.UPbit는 단순한 자동매매 봇이 아닙니다.  
**오픈소스 기여 → 커뮤니티 신뢰 → 프리미엄 전환**의 선순환을 만드는 엔진입니다.

### 1.1 기여 (Contribution)

| 기여 영역 | 내용 | 대상 |
|-----------|------|------|
| **OSS 패키지** | maiupbit PyPI — 기술지표, 백테스트, 퀀트 전략 | 전 세계 개발자 |
| **실전 트레이딩 데이터** | 매매 기록 + 분석 근거 + 사후 평가 공개 | 투자자/연구자 |

```

```

### 3.2 자동매매 (07:00 + 19:00) 상세 플로우

```
[Step 1] 데이터 수집 (30초)
├── UPbit API: 가격, 거래량, 호가, 잔고
├── 기술지표: RSI, MACD, BB, Stochastic, SMA, EMA
└── 현재 포지션 확인

[Step 2] 퀀트 전략 시그널 (10초)
├── 시즌 필터 (할빙 사이클 강세/약세)
├── 변동성 돌파 (래리 윌리엄스, 최적 K값)
├── 듀얼 모멘텀 스코어
└── 멀티팩터 랭킹

[Step 3] Mnemo 지식 조회 (5초)
├── 과거 유사 시장 상황 검색
├── 이전 매매 결과 패턴 ("RSI 30 이하 매수 → 평균 +2.8%")
└── 외부 수집 뉴스/분석 컨텍스트

[Step 4] LLM 종합 분석 (20초)
├── Ollama qwen2.5:14b (기본)
├── 입력: 기술지표 + 퀀트 + 지식 컨텍스트
└── 출력: {decision, confidence, reason, risk_params}

[Step 5] 매매 결정
├── confidence >= 0.60 → 실행
├── confidence < 0.60 → hold (기록만)
├── 예상 수익 > 0.3% (수수료 대비) → 실행
└── 일일 손실 -5% 초과 → 매매 중단

[Step 6] 실행 + 기록
├── UPbit 시장가 주문
├── TradeJournal: 분석 근거 + 체결 결과 저장
└── JSON stdout 출력 (MAIBOT 크론 파싱)
```

### 3.3 사후 평가 (07:30) 상세

```
[매일 07:30]
1. 24h+ 경과한 미평가 매매 조회
2. 현재 가격 조회 → 수익률 계산
3. 판단 정확도 평가:
   - sell 후 가격 하락 → ✅ 올바름
   - sell 후 가격 상승 → ❌ 틀림
   - buy 후 가격 상승 → ✅ 올바름
   - buy 후 가격 하락 → ❌ 틀림
4. TradeJournal 업데이트
5. Obsidian 노트 사후 평가 섹션 갱신
```

### 3.4 Obsidian 동기화 (05:30) 상세

```
[매일 05:30]
1. 전일 매매 기록 → Obsidian 노트 생성
   경로: 01.PROJECT/16.M.AI.UPbit/trades/YYYY-MM-DD_HH-MM_SYMBOL_ACTION.md
   
2. 노트 구조:
   - YAML frontmatter (tags, date, symbol, action, confidence, pnl)
   - 매매 기록 (종목, 수량, 가격, 수수료)
   - AI 분석 근거 (RSI, MACD, LLM 판단, 이유)
   - Mnemo 지식 참조 (검색 결과, 요약)
   - 사후 평가 (24h 후 가격, 수익률, 정확도)

3. Mnemo daily_enrich (05:00)가 이 노트를 자동 파싱
   → 지식그래프에 매매 패턴 축적
   → 다음 매매 시 KnowledgeProvider가 과거 패턴 검색
```

     ↓ 과거 경험 → LLM 컨텍스트 → 더 나은 판단

     ↓ 수익률·승률·AI정확도 대시보드

## 5. 리스크 관리

| 규칙 | 값 | 근거 |
|------|-----|------|
| 1회 매매 최대 비중 | 자산의 10% (~₩15,000) | 소액이지만 원칙 유지 |
| 일일 매매 횟수 | 2회 (07:00 + 19:00) | 데이터 축적 vs 수수료 균형 |
| 일일 손실 한도 | -5% | 초과 시 당일 매매 중단 |
| LLM 최소 confidence | 0.60 | 미달 시 hold (기록만) |
| 최소 예상 수익 | 0.3% (왕복 수수료 0.1% × 3) | 수수료 대비 충분한 마진 |
| UPbit 최소 주문 | ₩5,000 | 거래소 제약 |
| --confirm 안전장치 | 크론 스크립트만 자동 실행 | 수동 매매는 항상 확인 필요 |

### 5.1 월간 수수료 시뮬레이션

```
하루 2회 매매 × 월 20일 = 40회
왕복 수수료: ₩152,414 × 0.001 = ₩152/회
월 총 수수료: ₩152 × 40 = ₩6,097 (자산의 4%)

→ 월 +4% 이상 수익이 나야 수수료 커버
→ hold 판단(매매 안 함)이 많을수록 수수료 절감
→ AI confidence 기반 필터링이 핵심
```
