MAIJINI@openclaw Operating Notes #2 — Claude Code + MAIBOT + maiupbit: The Full-Stack AI Dev Chain

Not just chat-driven coding. A three-layer AI pipeline where conversation becomes code, code becomes execution, and results flow back as conversation.

The Architecture That Matters

This is not a story about using Claude Code to write commits. That is table stakes. The real thing we built in the last 10 days is a full-stack AI development chain where three distinct AI layers talk to each other through a single Discord DM thread:

Claude Code <-> Discord DM <-> MAIBOT (OpenClaw) <-> maiupbit CLI <-> UPbit Exchange

Layer 1: Claude Code — the coding agent. Reads the codebase, writes implementations, runs tests, commits code.

Layer 2: MAIBOT (OpenClaw/MAIJINI) — the orchestrator. Receives Discord DM messages, routes them to Claude Code, manages context, and can invoke tools on the host machine.

Layer 3: maiupbit CLI — the quant execution engine. A Click-based CLI with 13 subcommands that talks to UPbit API, runs momentum models, generates newsletters, and outputs everything as JSON.

The key insight: each layer was designed to be consumed by the layer above it. The CLI outputs JSON because MAIBOT needs machine-readable results. MAIBOT routes through Discord because Claude Code works through conversation. The whole chain is conversational at the top and computational at the bottom.

Why Three Layers, Not One

You might ask: why not just have Claude Code call UPbit directly?

Because each layer solves a different problem:

  • Claude Code is brilliant at understanding codebases and generating implementations, but it does not persist. Each session starts fresh. It cannot run a weekly cron job.
  • MAIBOT persists. It runs 24/7 on a Windows PC as an OpenClaw gateway. It has memory, browser control, file access, and scheduling. But it is an orchestrator, not a coding specialist.
  • maiupbit CLI encapsulates domain logic. Dual momentum calculations, multi-factor scoring, position sizing, risk thresholds — these need to be deterministic and testable, not regenerated by an LLM each time.

The three-layer architecture means each component does what it is best at. Claude Code writes the CLI. MAIBOT runs the CLI. The CLI runs the math.

What the Chain Produced in 10 Days (March 10-20)

1. cli/maiupbit.py — Built for Agent Consumption (375 LOC)

This is not a CLI built for humans typing in terminals. It is a CLI built for MAIBOT to invoke.

The docstring says it all: "MAIJini orchestration."

13 subcommands, every one outputting JSON by default:

  • maiupbit quant — full dual-momentum + multi-factor pipeline
  • maiupbit newsletter — generate the weekly AI Quant Letter
  • maiupbit status — portfolio state and P&L
  • maiupbit backtest — historical strategy performance
  • maiupbit journal — daily trade decision log

Why JSON? Because when MAIBOT calls maiupbit quant, it needs to parse the output, decide whether to trade, format a newsletter, and report back through Discord. Human-readable tables are useless to an agent. JSON is the agent's native language.

A typical chain looks like this:

  1. I send a Discord DM to MAIBOT: "Run this week's quant analysis"
  2. MAIBOT invokes maiupbit quant --output json
  3. The CLI fetches UPbit data, runs momentum models, returns structured results
  4. MAIBOT formats the results into the AI Quant Letter
  5. I get back the formatted newsletter in Discord, ready for Substack

2. Local LLM in the Chain: Qwen3:8b + GPU Layer Control

The quant pipeline includes a local LLM (Qwen3:8b via Ollama) for market analysis narrative. This is Layer 3 calling its own AI sub-layer — the CLI invokes the local model to generate prose around the numbers.

Problem: the 8B model was eating all GPU memory and causing CUDA fragmentation, crashing overnight runs.

Solution discovered through the chain itself (Claude Code suggested it via Discord, MAIBOT tested it):

OLLAMA_GPU_LAYERS=20

By limiting GPU layer offloading to 20 (out of ~32), we keep enough VRAM headroom for other tasks. This single environment variable eliminated the OOM crashes.

3. Newsletter Automation Pipeline

The AI Quant Letter now has a documented automation architecture that leverages all three layers:

  • D010 — Content generation pipeline: maiupbit CLI produces quant data, Qwen3 narrates, MAIBOT formats
  • D011 — Distribution pipeline: n8n webhook triggers MAIBOT, which calls the CLI, formats output, publishes to Substack API
  • GitHub Actions — Weekly cron trigger for the full pipeline

The goal: the chain runs autonomously every week. The quant model generates signals, the CLI packages them, MAIBOT publishes the newsletter and posts to social media — zero human intervention.

4. Regression Guard

When Claude Code writes new features through the chain, it could accidentally change quant parameters. The regression guard protects critical values:

  • Momentum lookback periods
  • Scoring weights
  • Risk thresholds
  • Position sizing multipliers

Every parameter change requires explicit justification. The guard runs in the test suite, which Claude Code executes before every commit.

What I Learned About AI Layering

1. Design each layer for the layer above it. The CLI outputs JSON because MAIBOT consumes it. MAIBOT speaks Discord because Claude Code works through conversation. Every interface is designed for its consumer, not its creator.

2. Orchestration is the hard layer. Claude Code is powerful. The CLI is straightforward. But MAIBOT — the middle layer that connects them, maintains state, handles errors, manages scheduling — that is where the real complexity lives.

3. The bottleneck moved from execution to thinking. With three AI layers handling the mechanical work, the limiting factor is "what should we build next?" not "how do I implement it?" The chain converts intent into deployed code faster than I can generate intent.

4. Local LLMs fit naturally as sub-layers. Qwen3:8b is not smart enough to orchestrate the whole pipeline, but it is perfect as a prose generator inside the CLI. Each layer can have its own AI sub-components at the right capability level.
